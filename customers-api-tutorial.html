Manage a customer record with the Customers API

A sample developer tutorial showing a simple customer lifecycle workflow across create, retrieve, update, and delete operations.

Overview

Sample content · Fictional API

This is a sample tutorial created for portfolio purposes. The API, URLs, and example data are fictional and are included only to demonstrate structure, flow, and writing style.

In this tutorial, you create, retrieve, update, and delete a customer record by using the /api/1.0/customers endpoint and the permanent customer id returned by the API.

Before you start

Requirements

Make sure you have the following:

  • An API access token.
  • Your environment base URL.
  • A tool for sending HTTP requests, such as curl or Postman.
  • Access to the /api/1.0/customers endpoint.

In the examples below, replace {base_url} and {access_token} with your own values.

1. Create a customer

POST /api/1.0/customers

Send a POST request to create a new customer record. The response returns a permanent id that you use in later requests. 

curl -X POST "{base_url}/api/1.0/customers" \
-H "Authorization: Bearer {access_token}" \
-H "Content-Type: application/json" \
-d '{
  "full_name": "John Doe",
  "email": "john@doe.com"
}'
const response = await fetch(`${baseUrl}/api/1.0/customers`, {
  method: "POST",
  headers: {
    Authorization: `Bearer ${accessToken}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    full_name: "John Doe",
    email: "john@doe.com"
  })
});
const customer = await response.json();

Example response:

{
  "id": "9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh",
  "full_name": "John Doe",
  "email": "john@doe.com",
  "created_at": "2022-06-13T12:03:39.979397Z",
  "updated_at": "2022-06-13T12:03:39.979397Z"
}

2. Retrieve a customer

GET /api/1.0/customers/{id}

Send a GET request to retrieve the customer record by using the id returned in the previous step. 

curl -X GET "{base_url}/api/1.0/customers/9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh" \
-H "Authorization: Bearer {access_token}"
const response = await fetch(
  `${baseUrl}/api/1.0/customers/9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh`,
  {
    method: "GET",
    headers: {
      Authorization: `Bearer ${accessToken}`
    }
  }
);
const customer = await response.json();

Example response:

{
  "id": "9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh",
  "full_name": "John Doe",
  "email": "john@doe.com",
  "created_at": "2022-06-13T12:03:39.979397Z",
  "updated_at": "2022-06-13T12:03:39.979397Z"
}

3. Update a customer

PATCH /api/1.0/customers/{id}

Send a PATCH request to update selected fields on the customer record. In this example, you update the customer's name and add a business name. 

curl -X PATCH "{base_url}/api/1.0/customers/9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh" \
-H "Authorization: Bearer {access_token}" \
-H "Content-Type: application/json" \
-d '{
  "full_name": "John Smith",
  "business_name": "Blahblah Ltd"
}'
const response = await fetch(
  `${baseUrl}/api/1.0/customers/9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh`,
  {
    method: "PATCH",
    headers: {
      Authorization: `Bearer ${accessToken}`,
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      full_name: "John Smith",
      business_name: "Blahblah Ltd"
    })
  }
);
const customer = await response.json();

Example response:

{
  "id": "9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh",
  "full_name": "John Smith",
  "business_name": "Blahblah Ltd",
  "email": "john@doe.com",
  "created_at": "2022-06-13T12:03:39.979397Z",
  "updated_at": "2022-06-13T12:10:13.979397Z"
}

Only the fields included in the request are updated. Other fields, such as email, remain unchanged.

4. Delete a customer

DELETE /api/1.0/customers/{id}

Send a DELETE request to remove the customer record.

curl -X DELETE "{base_url}/api/1.0/customers/9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh" \
-H "Authorization: Bearer {access_token}"
await fetch(
  `${baseUrl}/api/1.0/customers/9c7c97a8-ac45-bh78-8b38-26a74fdf1fgh`,
  {
    method: "DELETE",
    headers: {
      Authorization: `Bearer ${accessToken}`
    }
  }
);

A successful request returns:

204 No Content

This confirms that the customer record has been deleted.

Result

End-to-end workflow

You have created a customer, retrieved it by id, updated selected fields, and deleted the record. This is a basic end-to-end workflow for managing customer data with the Customers API.

Notes

Implementation details

  • Use the same customer id throughout the workflow.
  • Include the Authorization header in every request.
  • Use Content-Type: application/json for requests with a body.
  • Use PATCH when you want to update selected fields instead of replacing the full object.
