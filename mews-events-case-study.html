Case study

Mews Events

Documentation architecture for a newly acquired module used to manage quotes for hotel-hosted events.

Back to portfolio Open documentation hub

Overview

Information architecture · Help center content

Mews Events, originally Quotello, was brought into the Mews product ecosystem as a module for event quotes and related workflows. Before release, the documentation existed only as fragmented internal notes in SharePoint, written mainly for support use and not ready for public help content.

What I did

Structure · Writing · Localization readiness

I designed the documentation architecture for the new product area, including navigation nodes, internal CMS structure in Salesforce, page templates, cross-linking, tagging, and content writing. The goal was to align the module with the existing Mews help center model and make the content easier to find, localize, and maintain.

Approach

Audit · Workflow-based structure · Terminology alignment

I worked from limited original material, product demos, stage testing, and discussions with product, UX, onboarding, support, and development teams. The final structure combined UI-based navigation with task-based workflows and clearer glossary alignment, while also linking key articles back into the product UI.

Result

Release-ready documentation

The result was a public-facing documentation hub ready at release, with a clearer structure for users, better support for onboarding, and a solid base for future content improvements, localization, and AI consumption.
